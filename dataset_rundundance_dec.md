# 昇腾算子生成思维链冗余操作规避指导

## 一、技能描述

本文基于TopKTopP昇腾AscendC算子生成思维链的冗余操作分析，明确各类冗余场景的规避标准、操作步骤及注意事项，用于规范算子生成全流程，减少无效开销、精简数据体积、提升执行效率与可读性，适用于所有昇腾算子（基于Claude工具链）的思维链设计与执行，为Agent Skill算子生成提供明确避错依据。

---

## 二、Agent全流程规避执行标准

### 1. 思维链结构标准（Agent必遵循）

```json
{
  "task": "生成XXX昇腾AscendC算子",
  "global_config": {
    "npu": 0,
    "op_file": "算子文件路径",
    "output_dir": "输出目录路径",
    "sessionId": "会话ID"
  },
  "used_skills": ["所需技能1", "所需技能2", ...],
  "execution_steps": [
    { "phase": "0", "name": "参数确认", "operation": "参数校验", "status": "done" },
    { "phase": "1", "name": "环境准备", "operation": "合并调用Read+mkdir工具", "status": "done" },
    { "phase": "2", "name": "用例精简", "operation": "调用case-simplifier技能", "status": "done" },
    ...
  ],
  "metadata": {
    "usage": { "input_tokens": 0, "output_tokens": 0 },
    "stop_reason": "complete"
  }
}
```

### 2. Agent执行原则（强制遵循）
1. 必要性原则：所有步骤、字段、工具调用均需与算子生成核心逻辑相关，无关内容一律删除。
2. 唯一性原则：全局固定配置、参数解析、元数据统计仅保留1次，不重复出现。
3. 高效性原则：无依赖的工具调用尽量合并，减少交互轮次；技能按需加载，避免无效消耗。
4. 可读性原则：冗余信息删除后，思维链流程清晰、步骤明确，关键操作无遮挡。

---

## 三、核心冗余场景及规避方法（Agent Skill执行规范）
### 1. 固定字段冗余（优先级：最高）
**冗余场景**：思维链所有消息节点重复携带无变化的全局字段（如sessionId、userType、cwd、version等），无任何业务增量信息。
**Agent执行要求（规避方法）**：
- 抽取所有全局固定配置，单独建立「全局配置」模块，仅在思维链开头声明1次，后续步骤不再重复携带。
- 全局配置需包含的核心字段：npu、op_file、output_dir、sessionId，其余无变化的元字段（如version、gitBranch）可省略或单独归档。
- 示例：单独新增「全局配置」章节，集中存放固定信息，步骤中仅引用必要参数。
**禁忌（Agent禁止操作）**：禁止在每个工具调用、步骤描述中重复打印全局固定字段。

---

### 2. 元数据与Token统计冗余
**冗余场景**：所有assistant消息重复输出usage（输入输出Token统计）、stop_reason、stop_sequence等无变化的元数据。
**Agent执行要求（规避方法）**：
- 仅在思维链执行结束后，统一输出1次Token统计、执行状态等元数据，无需在每个步骤节点打印。
- 若无需统计Token消耗，可直接省略usage字段；stop_reason、stop_sequence仅在状态变化时打印（如从tool_use变为complete）。

---

### 3. 参数重复解析冗余
**冗余场景**：算子核心参数（npu、op_file、output_dir）在用户指令、思考阶段、参数确认表格中重复解析、重复声明。
**Agent执行要求（规避方法）**：
- 参数确认仅保留1个环节，优先在「全局配置」或「Phase 0：参数确认」中一次性完成解析与声明，后续步骤直接引用，不重复解析。
- 取消参数重复打印，无需在思考文本、步骤描述中再次重复参数值。

---

### 4. 工具调用拆分冗余
**冗余场景**：无依赖关系的工具调用被拆分（如先调用Read读取算子文件，紧接着调用Bash创建输出目录），增加交互轮次。
**Agent执行要求（规避方法）**：
- 合并无依赖的工具调用，将多个独立操作整合为1次工具调用，减少交互耗时。
- 示例：将「Read读取文件」与「Bash创建目录」合并为单次工具调用，一次性执行两个操作。
**Agent注意事项**：有依赖关系的工具调用（如先完成用例精简，再进行算子转译）不可合并，需按流程顺序执行。

---

### 5. 无效技能列表加载冗余
**冗余场景**：一次性加载全量技能列表（如9个技能），但实际仅用到少数几个与当前算子任务相关的技能，其余技能无效加载。
**Agent执行要求（规避方法）**：
- 按需加载技能，仅加载当前算子生成任务所需的技能（如TopKTopP算子仅加载ascendc-translator、case-simplifier等4个相关技能）。
- 取消全量技能列表加载，思维链中仅声明并调用所需技能，无关技能不提及、不加载。

---

### 6. 无业务价值节点冗余
**冗余场景**：存在与算子生成核心逻辑无关的节点（如会话调度的enqueue/dequeue操作），仅为调度记录，无实际业务意义。
**Agent执行要求（规避方法）**：
- 删除所有与算子生成无关的节点，仅保留核心执行步骤（参数确认、环境准备、用例精简、算子实现、性能分析等）。
- 会话调度、日志记录等非核心内容，单独归档，不纳入思维链核心流程。

---

### 7. 思维链与文本重复冗余
**冗余场景**：thinking思考字段已完成阶段规划，后续text文本又重复打印相同阶段内容，内容高度重叠。
**Agent执行要求（规避方法）**：
- 保留思考阶段的核心规划，删除重复的文本描述，避免阶段内容重复打印。
- 思维链思考内容以“规划步骤”为主，步骤执行描述以“实际操作+结果”为主，两者不重复、不重叠。

---

### 8. 非必需备份操作冗余
**冗余场景**：自动化算子生成流程中，添加非必需的文件备份操作（如model.py.bak），属于防御性冗余，无实际必要。
**Agent执行要求（规避方法）**：
- 自动化流程中取消非必需的文件备份操作，仅在手动操作、高风险操作场景下保留备份。
- 若需保留备份，可将备份操作纳入“异常处理”模块，不单独作为核心步骤。

---

## 四、Agent执行注意事项
- 规避冗余不影响核心功能，需确保算子生成的正确性、完整性，不可因精简而省略必要步骤（如用例验证、性能分析）。
- 不同类型昇腾算子（如TopK、Softmax）冗余场景一致，可直接复用本指导方法。
- 若后续工具链升级、流程变更，需同步更新冗余规避方法，确保指导的适用性。
- 思维链生成后，Agent需自动检查是否存在未规避的冗余场景，重点核查固定字段、重复解析、无效技能等核心冗余点。
